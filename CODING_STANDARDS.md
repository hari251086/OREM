# OREM — Coding Standards

Rules for code contributed to this repository, independent of any single
issue or version. Architectural rationale lives in `ARCHITECTURE.md`;
build/test mechanics live in `README.md`. This file is for OREM-specific
standing rules — cross-repo rules (4-core cap, C: drive, git workflow, no
Co-Authored-By, etc.) live once in `GitHub\CLAUDE.md`, not duplicated here.

No OREM-specific standing rules beyond the shared ones yet — this file is
the place to add them as they come up, rather than folding them into
`ARCHITECTURE.md` or `README.md`.
